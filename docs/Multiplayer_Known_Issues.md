# 멀티플레이 미해결 기록

기록일: 2026-06-22

현재 2~4인 로비 입장과 게임 맵 전환을 작업 중이다. 좌석 카메라 배정은 일부 실행에서 정상적으로 보였지만, 반복 실행 시 일부 클라이언트가 건물 바깥/기본 시점에 남는 현상이 다시 발생한다. 따라서 멀티플레이 게임 루프는 아직 플레이 가능한 상태가 아니다.

## 다음 작업에서 재현·수정할 항목

1. 게임 맵 좌석 배정 안정화
   - 증상: 방장만 실내 테이블 시점으로 이동하거나, 일부 참가자가 건물 바깥의 기본 Pawn/PlayerStart 시점에 남는다.
   - 기대: 모든 클라이언트가 자신의 `PlayerState.ShowDownSlot`에 따라 `MP_SeatCamera_1`부터 `MP_SeatCamera_4`까지의 해당 카메라를 항상 사용한다.
   - 확인: 2인, 3인, 4인 각각을 연속으로 여러 번 시작해도 모든 창이 테이블 시점으로 들어오는지 검증한다.

2. 멀티플레이 카드 게임 루프
   - 증상: 게임 맵에 들어가도 카드가 각 플레이어 좌석/카메라 앞에 안정적으로 표시되지 않고, 카드 선택·전달·이마 카드 공개·베팅을 정상 진행할 수 없다.
   - 기대: 서버가 덱/카드 소유/턴을 판정하고, 각 플레이어에게는 싱글플레이와 동일한 규칙(내 카드는 보지 않고 상대에게 줄 카드를 선택)으로 표시한다.

3. 채팅
   - 증상: 게임 중 채팅을 열거나 메시지를 주고받을 수 없다.
   - 확인 대상: `ChatWidgetClass` 생성 여부, 입력 모드 전환, 서버 RPC 및 GameState 메시지 복제.

4. 게임 중 나가기
   - 증상: `Esc` 확인창/나가기 동작이 실제 게임에서 사용할 수 없다.
   - 기대: `Esc` → 확인창 → 나가기/취소. 나가기는 해당 사용자를 허브로 이동시키고, 방장이 나가면 연결된 참가자도 허브로 이동시킨다.
   - 확인 대상: C++ 확인창 생성·뷰포트 표시, 입력 포커스, EOS 세션 정리와 클라이언트 이동 순서.

5. 마우스 입력
   - 증상: 카드 선택용 커서를 유지하면서도 시점 회전이 일관되게 동작하는지 재검증이 필요하다.
   - 기대: 활성화된 게임 창에서 별도 우클릭 없이 마우스 이동으로 시점을 회전하고, 카드 위에서는 커서로 클릭 선택할 수 있다.

## 현재 원칙

- 게임 시작 뒤 중간 입장은 허용하지 않는다.
- 퇴장은 허용하되 확인창을 거친다.
- 멀티플레이의 최종 판정 권한은 서버에 둔다.
- 플레이어 외형은 향후 `PlayerState`의 장착 스킨 ID와 분리된 복제 Avatar Actor로 구성한다.
